0: /**
0: 	List of error message identifiers.
0: 	This is the set of message identifiers. The message identifier
0: 	also encodes the SQLState as the first five characters.
0: 	StandardExceptions must be created using the static
0: 	StandardException.newException() method calls, passing in a
0: 	field from this class.
0: 	<BR>
0: 	The five character SQL State is obtained from a StandardException
0: 	using the zero-argument StandardException.getSQLState() method.
0: 	<BR>
0: 	The message identifier (ie. the value that matches a field in this class)
0: 	is obtained using the zero-argument StandardException.getMessageId() method.
0: 	<BR>
0: 	Thus if checking for a specific error using a field from this interface
0: 	the correct code is
0: 	<PRE>
0: 		if (se.getMessageId().equals(SQLState.DEADLOCK))
0: 	</PRE>
0: 	<BR>
0: 	A utility static method StandardException.getSQLState(String messageId)
0: 	exists to convert an field from this class into a five character SQLState.
0: 	<P>
0: 
0: 	<P>
0: 	The SQL state of an error message dictates the error's severity.
0: 	The severity is determined from the first two characters of the
0: 	state if the state is five characters long, otherwise the state
0: 	is expected to be 7 characters long and the last character determines
0: 	the state. If the state is seven characters long then only the first
0: 	five will be seen by the error reporting code and exception.
0: 	<BR>
0: 	If the state is 9 characters long, the last two characters encode
0: 	an exception category, which Synchronization uses to determine whether
0: 	the error causes REFRESH to halt or to simply skip the failed transaction.
0: 	All 5 and 7 character states default to the ENVIRONMENTAL exception
0: 	category.
0: 	<BR>

0: 	<LI> <B> WARNING_SEVERITY </B>

0: 
0: 	<LI> <B> NO_APPLICABLE_SEVERITY </B>

0: 	This is the set of message identifiers. The message identifier
0: 	also encodes the SQLState as the first five characters.
0: 	StandardExceptions must be created using the static
0: 	StandardException.newException() method calls, passing in a
0: 	field from this class.
0: 	<BR>
0: 	The five character SQL State is obtained from a StandardException
0: 	using the zero-argument StandardException.getSQLState() method.
0: 	<BR>
0: 	The message identifier (ie. the value that matches a field in this class)
0: 	is obtained using the zero-argument StandardException.getMessageId() method.
0: 	<BR>
0: 	Thus if checking for a specific error using a field from this interface
0: 	the correct code is
0: 	<PRE>

0: 	</PRE>

0: 	exists to convert an field from this class into a five character SQLState.

0: 	The SQL state of an error message dictates the error's severity.
0: 	The severity is determined from the first two characters of the
0: 	state if the state is five characters long, otherwise the state
0: 	is expected to be 7 characters long and the last character determines
0: 	the state. If the state is seven characters long then only the first
0: 	five will be seen by the error reporting code and exception.
0: 	<BR>
0: 	If the state is 9 characters long, the last two characters encode
0: 	an exception category, which Synchronization uses to determine whether

0: 	All 5 and 7 character states default to the ENVIRONMENTAL exception
0: 	category.
